## Background: Why This Work Exists

The 117 resort records in ResortIQ were generated by an AI scraper that extracted factual data (names, locations) from public brand websites. The data has three problems:

**Problem 1 — Templated content:** Every resort description follows the same formula. Every Hilton property has the same amenities list, same phone number (+1-800-932-4482), same 4.3 rating. This is brand-level data stamped 117 times, not per-resort data.

**Problem 2 — IP risk:** Before any resort description is exposed via a public API or displayed to users, it must be written in RAV's own voice — original, factual, not copied from any brand website.

**Problem 3 — Missing structured data:** Addresses are "City, State, Country" not real addresses. Airport codes are placeholder text. These need to be flagged and queued for enrichment.

## Phase 2: Implementation (after user story approval)

### Deliverable 1: Data quality audit script
Create `scripts/audit-resort-data-quality.ts`

This script should:
1. Connect to Supabase using service role key from `.env.local`
2. Fetch all resorts with their unit types
3. For each resort, check and flag:
   - Description matches template pattern? → flag `TEMPLATE`
   - `nearby_airports` contains "Local airport"? → flag `PLACEHOLDER`
   - `contact.phone` equals "+1-800-932-4482"? → flag `GENERIC_PHONE`
   - `guest_rating` equals 4.3 (same for all of same brand)? → flag `BULK_RATING`
   - `latitude`/`longitude` null? → flag `NO_COORDS`
   - `postal_code` null? → flag `NO_POSTAL`
   - Description under 80 chars? → flag `SHORT_DESCRIPTION`
4. Output report to `docs/features/mdm-resort-data/data-quality-report.md`:
   - Summary table: total records, % with each flag
   - Per-brand breakdown
   - Records below score 40 (needs immediate attention)
   - Records above score 70 (acceptable)

interface ResortAuditResult {
  id: string;

const PLACEHOLDER_AIRPORT = 'Local airport';

### Deliverable 2: Description generator script
Create `scripts/generate-resort-descriptions.ts`

**IMPORTANT: Generate descriptions from structured data only — no web scraping, no copying from brand websites.**

Description formula:
```
[RESORT_NAME] is a [BRAND_DISPLAY_NAME] vacation ownership resort located in [CITY], [STATE_ABBR].
[If unit types: The resort offers [count] accommodation types, from [min]-bedroom to [max]-bedroom units, sleeping up to [max_occupancy] guests.]
[If amenities: Guests enjoy resort amenities including [top 3-4 amenities].]
[If policies: Standard check-in is at [check_in_time].]
```

Rules:
- Never use superlatives ("world-class", "premium", "luxury") — brand claims, not facts
- Never reference Hilton/Marriott/Disney marketing language
- Write in third person ("Guests enjoy", "The resort offers")
- Keep descriptions 80–200 words
- No word in descriptions should appear in BRAND-LOCK.md prohibited terms

The script should:
1. Read all resorts from Supabase
2. Generate descriptions from structured data
3. Write to preview file `docs/features/mdm-resort-data/generated-descriptions-preview.json` — do NOT update the database yet
4. Flag resorts where structured data is too sparse for meaningful description
5. Print: "X descriptions generated, Y need manual review"
